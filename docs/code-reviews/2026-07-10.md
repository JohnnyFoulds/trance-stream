## Executive Summary

The synthesis core has three runtime bugs that corrupt audio output unconditionally: phantom sidechain ducking from IIR residual amplification, a permanently saturated FM depth on the pulse instrument, and a hey-angel song rendered at 55% gain through the v3 path. The test suite has significant structural weaknesses — multiple tests that pass trivially when the feature under test is completely broken — and the hey-angel arrangement has diverged substantially between `hey_angel_cover.py` and `trance_stream_v3.py --style hey_angel`, with different instrument classes, glide rates, and missing bass events. Recommended priority: fix the three audio-corruption bugs first, then address the test-coverage holes, then tackle the architecture divergence.

## Findings

### Critical — fix before merging

**[CORRECTNESS] Sidechain IIR residual amplified to full duck depth on every post-kick block** `synth/effects.py:327`
> The per-block peak normalization divides the smoothed envelope by `max(kick_env_smooth.max(), 1e-9)`. When a kick fires in bar 1, the IIR state decays but never reaches exactly zero. At the start of bar 2 the residual is ~6e-6; line 325 computes `peak = 6e-6`, line 326 divides by `max(6e-6, 1e-9) = 6e-6`, rescaling the residual to 1.0. Line 328 then computes `gain = 1 - 0.6 * 1.0 = 0.4` — full duck depth — even though no kick fired. The `1e-9` floor prevents divide-by-zero but not sub-threshold amplification. Fix: add a threshold guard before normalization, e.g. `if peak < 1e-4: kick_env_norm = np.zeros_like(kick_env_smooth)` instead of normalizing.
> Failure scenario: With `sr=44100`, `attack_s=0.16`, `depth=0.6`, a single kick in bar 1 produces `gain[0]=0.4` at the start of every subsequent bar for the lifetime of the session.

**[CORRECTNESS] FM depth is always 0.5 — session-arc idiom is never realized** `instruments/pulse.py:39`
> `fm_depth = 0.5 * (total_steps / max(total_steps, 1))` simplifies to `0.5 * 1.0 = 0.5` for every call where `total_steps >= 1`. The comment on line 38 says "FM depth increases with session time (fm(time) idiom): 0 → 0.5" — this is false. The denominator equals the numerator for all positive values, so the ratio is always 1.0. Only the unreachable `total_steps=0` case gives `fm_depth=0`. Fix: replace with `fm_depth = 0.5 * (step / max_session_steps)` where `max_session_steps` is a fixed expected session length.
> Failure scenario: The texture starts maximally FM-modulated from step 1 and never changes, contradicting the design intent for every session of any length.

**[CORRECTNESS] `gain_arc()` with `filter_pb_bar=9999` renders hey-angel at ~55% gain** `song/arcs.py:231`
> `build_hey_angel_song()` sets `filter_pb_bar=9999`. `SongRenderer._render_bar()` unconditionally calls `gain_arc(bar, song)` and multiplies both mix channels by the result. The early-return guard fires only when `bar >= ramp_end`, which never happens in a 128-bar song. The ramp formula `0.55 + 0.45*(bar/9999)` yields 0.550 at bar 0 and only 0.556 at bar 127. Fix: add a sentinel check in `build_hey_angel_song()` to set `filter_pb_bar` to the actual song length, or add an explicit override in `SongRenderer` for the hey-angel style.
> Failure scenario: `python trance_stream_v3.py --style hey_angel` outputs audio at 55% amplitude with no warning.

**[CORRECTNESS] `chord_state_at()` phase jump at pullback transition** `song/arcs.py:257`
> Before pullback: `weights=[3,1,3,1]`, `cycle_len=8`. After pullback: `weights=[2,1,2,1]`, `cycle_len=6`. No phase-alignment offset is applied. At `filter_pb_bar=64`: bar 63 → chord_idx=3; bar 64 → chord_idx=2. A simulation over all 32 possible pullback bars confirms 24 of them (75%) produce a discontinuous chord index jump. Fix: return a phase offset from `chord_state_at()` that aligns `bar % new_cycle` to the current chord position at the transition bar.
> Failure scenario: 75% of seeds produce an audible wrong chord at the pullback transition.

**[CORRECTNESS] `notearp_to_midi()` crashes with `ZeroDivisionError` on empty chord** `song/renderer.py:514`
> `chord_to_midi([])` returns `[]`. The loop reaches a `SA_NOTEARP_PATTERN` entry `>= 0` and executes `chord_midi[idx % 0]`, raising `ZeroDivisionError`. No guard exists. Fix: add `if not chord_midi: return []` at the top of `notearp_to_midi()`.
> Failure scenario: Any call path that produces an empty chord degrees list crashes the renderer.

**[CORRECTNESS] Hey-angel bass in `SongRenderer` missing second half-bar** `song/renderer.py:279`
> Lines 272–302 render only steps 0, 4, 6 (first half). Steps 8, 12, 14 present in `hey_angel_cover.py`'s `BASS_PATTERN` are never rendered. No guard, fallback, or alternative code path fills the second half.
> Failure scenario: Sub-bass energy under the second kick at step 8 is completely absent in the v3 path, making every second beat audibly thinner.

**[CORRECTNESS] `SupersawPad.render()` hardcodes 140 BPM — trancegate drifts at any other tempo** `instruments/pad.py:91`
> `samples_per_bar()` called with no arguments (BPM=140 hardcoded). At 138 BPM, the error is 1095 samples (~24.83 ms) per bar; after 32 bars the trancegate is ~795 ms out of phase with the kick. `instruments/lead.py` already received the `samples_per_bar` parameter fix. Fix: add `samples_per_bar: int | None = None` parameter to `SupersawPad.render()`.
> Failure scenario: The pad pump audibly de-syncs from the kick after ~32 bars at 138 BPM.

## Architecture Observations

**Hey-angel arrangement split across two incompatible entry points.** `hey_angel_cover.py` uses `SmoothLead` (`render(midi_note: int)`) while `trance_stream_v3.py --style hey_angel` uses `AcidLead(character='smooth')` (`render(midi_notes: list)`). Different synthesis classes, incompatible signatures, different glide rates, missing bass events, different gain. Any work on the hey-angel sound must be applied twice with no type-system link enforcing consistency. Designate one path as canonical and delete the other, or converge on a shared instrument class with a unified `render()` API.

**Three concurrent entry-point scripts with no deprecation notices.** `trance_stream.py` and `trance_stream_v2.py` import `sounddevice` unconditionally (unguarded `ImportError` on systems without it); v3 wraps imports in `try/except`. Bug fixes to v3's instrument stack do not flow to v1/v2. Fix: add deprecation headers to v1 and v2 pointing to v3, guard their `sounddevice` imports.

**Hey-angel MIDI note constants duplicated across files.** G1=43, F2=53, MELODY_START=60, MELODY_END=54, E5=76 appear as local variables in `renderer.py` and as module-level constants in `hey_angel_cover.py` (F2=53 appears only as an inline literal in `BASS_PATTERN`). `song/theory.py` defines none of them. A measurement revision requires updating two files with no static link connecting them.

**Monotonic stage-bars ordering pass duplicated verbatim** at `song/builder.py:231–237` and `284–290`. A new stage key (e.g. `pluck_on`) must be added to both copies independently or one arc_shape path silently loses the monotonic guarantee.

**CLAUDE.md measurement-first discipline not enforced.** The FM depth bug (`pulse.py:39`) and the sidechain depth bug (`effects.py:327`) both contradict stated targets from `prebake.strudel` measurement. Running the values through `research/strudel_debug.html` as required by CLAUDE.md would have caught both immediately.

## Test Coverage Summary

**Well-covered:** `synth/envelopes.py`, `synth/filters.py` (lpf only), `instruments/kick.py`, `song/builder.py` arc ordering, `tools/visualiser.py` border/color modes.

**Under-tested / structurally weak:**
- `synth/effects.py` — no test for sidechain residual amplification; reverb has only smoke tests
- `song/arcs.py` — `chord_state_at()` phase alignment across pullback untested; `gain_arc()` with sentinel value untested
- `song/renderer.py` — `notearp_to_midi()` with empty chord untested; `write_midi()` content untested; hey-angel bass second half-bar untested
- `instruments/pad.py` — BPM dependency of trancegate phase untested; trancegate RMS variation test has vacuous-pass bug
- `instruments/bass.py` / `instruments/lead.py` — `portamento_s` float value behavior untested; `lpf2` state continuity untested
- `tests/test_hey_angel_synthesis.py` — three of four tests are non-silence checks masquerading as feature tests

**Most important missing tests (impact order):**
1. Sidechain residual: two consecutive bars, kick in bar 1 only; assert `gain[0]` in bar 2 `> 0.95`
2. `chord_state_at()` phase continuity: assert `chord_idx(pb_bar-1) == chord_idx(pb_bar)` for representative pullback bars
3. `notearp_to_midi()` with empty input: assert returns `[]` without raising
4. `lpf2` state continuity: single-call vs 64-segment-call output within `1e-5`
5. Hey-angel bass completeness: assert non-zero energy in second half-bar for v3 path
6. `write_midi()` output contains note events
7. Pluck brightness (requires VCF implementation first): assert `c_early > 2000`, `c_late < 1800`
